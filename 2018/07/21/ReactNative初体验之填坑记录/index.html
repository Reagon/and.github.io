ReactNative填坑记录

最近在自学React Native，真的是一步一个坑，如果不记录一下很容易忘记，前一天刚配置好的环境，今天建个新项目又开始重复昨天的问题，还要找一遍解决问题的办法，第一时间找到了还好，怕的是怎么找都找不到了。。。，浪费了很多宝贵的时间，所以还是记录一下，方便以后查看。

React Native现在最新版本是0.56，在初始化项目的时候默认使用的就是最新版本，建立项目到运行项目的步骤是：

react-native init projectName

注意：以下新建项目的步骤亲测无效

初始化项目的时候看到别人的博客上和书上说如果想建立旧版本的项目可以使用

react-native init demo --verbose --version 0.55.4

或者rninit工具，先安装rninit工具，在命令行输入：

npm install -g rninit

安装好之后，通过rninit命令初始化指定版本的RN项目，命令是：

rninit init demo -source react-native@0.55.4

看到这个的时候因为已经找了好久解决办法，已经晚上一点了，真的是又困又累，所以心里还暗喜了一下，心想终于有办法了，但是，但是，但是事实证明空欢喜一场，因为运行项目的时候还是个大红屏幕，去项目的根目录下查看package.json文件发现React Native版本还是0.56.0，所以这两种办法根本没用。


初始化项目后还要将电脑的8081端口反向代理到测试机上，如果电脑只连接一个Android设备，可以使用简化命令：

adb reverse tcp:8081 tcp:8081

如果一个设备以上，使用adb devices查看设备列表，然后指定设备名称:

adb -s 设备名 reverse tcp:8081 tcp:8081

正常的话执行这条命令后什么也不会出现，如果提示adb命令不是内部命令或外部命令，配置一下环境变量即可。

运行项目：

react-native run-android

在windows电脑上0.56版本的项目一直是上面那种红色的屏幕，查了别人的解决办法是降级react-native版本到0.55.4，在工程目录下使用命令：

npm install --save react-native@0.55.4

然后

react-native upgrade

然后一路选择： Y ，回车

这个时候去项目目录下查看package.json文件react-native版本变成0.55.4了，现在运行项目，依然是个红屏幕，然后去工程目录下鼠标右键，删除 node_modules文件夹，然后再命令行执行：

npm install

再次 react-native run-android 运行项目可能还是个红屏幕，报错类似 Error: Plugin 0 specified in… ，这个时候执行命令：

npm install --save-dev babel-preset-react-native@2.1.0

再次 react-native run-android ，哇塞，终于不是红屏幕了（手动鲜花，掌声）！

这个项目是我用公司的Mac建立的，rn版本号为0.56，上传到了github，回到家里用的是windows 10 系统，clone下来后运行报错，所以按照网上说的方法降级到0.55.4，经过做一遍又一遍的修改删除重新clone一番折腾后终于运行成功了。

在运行项目的时候还有个错误是找不到 android\gradle\wrapper\gradle-wrapper.properties 这个文件，用android studio打开一下这个android工程就自动生成那个文件了，注意不是运行整个rn项目，是单独的android项目。
